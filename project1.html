Thesis Capstone: The Adventures of Tom: A Python-based Interactive Assessment Game for BSCS Freshmen Students of Colegio de San Gabriel Arcangel (CDSGA)

Source Code

We developed "The Adventures of Tom," an interactive assessment game aimed at quantifying students' proficiency and assisting them in their learning and review process. It encompasses essential features such as quests, lesson delivery, a program tracker, review questions, and assessments that provide quick results upon completion. The game specifically focuses on the subject of Computer Programming 1, covering topics related to programming concepts, logic formulation, and an introduction to the Java programming language and its features.

As a programmer for "The Adventures of Tom," a Python-based interactive assessment game for BSCS freshmen students of Colegio de San Gabriel Arcangel (CDSGA), I focused on developing core mechanics such as player movement, interactive elements, and creating a captivating game world. I leveraged Python's versatility to ensure smooth navigation between different timelines using a time portal. Additionally, I contributed to integrating engaging features like the main menu, quest log, and lesson log for a user-friendly experience. Working with the team, we utilized various tools and technologies to enhance functionality and visual appeal, strengthening my skills in Python programming and game development.

We are incredibly honored and proud to announce that our thesis capstone project has been awarded the prestigious title of 'Best Thesis' in our department at Colegio De San Gabriel Arcangel. This recognition serves as a testament to the dedication, hard work, and passion that we poured into our research and project. We would like to extend our heartfelt gratitude to our advisors, mentors, and the entire faculty who supported and guided us throughout this challenging journey. This award is a culmination of countless hours of research, analysis, and collaboration, and it motivates us to continue striving for excellence in our academic and professional pursuits. We are truly grateful for this recognition and look forward to contributing more to our field in the future.

Here are some of the key tools we utilized in the development of "The Adventures of Tom":

  • Python: The core language powering the game's logic and mechanics.

  • Pygame: A powerful game development framework in Python, enabling us to create an immersive and interactive gaming experience.

  • Tiled: A versatile map editor that allowed us to design and create the game's captivating maps and environments.

  • Aseprite: An excellent tool for pixel art creation and animation.

  • Photoshop: Another valuable tool for pixel art creation, contributing to the overall aesthetic of the game.

User's Manual of the System

Main Menu Interface

A main menu to welcome users; used to access login page, options, etc.

  1. This button will redirect the user to the login page when clicked.
  2. After logging in, clicking this button redirects the user to the game.
  3. This button redirects the user to the options menu.
  4. This button is used to quit the application.

Login Interface

To prevent unauthorized users from accessing the system; to prevent users from accessing panels that are not meant for them

  1. This textbox is designated for the entry of the user's username.
  2. This textbox is designated for the entry of the user's password.
  3. This button redirects the user to the respective window depending on the userlevel the account has.
  4. This button redirects the user to the registration page when clicked.
  5. This button redirects the user back to the main menu interface.

Register Interface

To help unregistered users create a user account to access the system.

  1. This textbox is designated for the unregistered user's chosen username.
  2. This textbox is designated for the unregistered user's chosen password.
  3. This textbox is designated for the entry of the unregistered user's first name.
  4. This textbox is designated for the entry of the unregistered user's last name.
  5. This dropdown menu is designated for the selection of the user's program.
  6. This dropdown menu is designated for the selection of the user's year level.
  7. This textbox is designated for the entry of the user's student number.
  8. This button allows the user to get registered.
  9. This button redirects the user back to the main menu interface.

Administrator Panel

This panel is exclusive to the administrator and is used to approve or declined newly registered accounts.

  1. This dropdown menu is for the selection of a pending account.
  2. This button allows the details of a pending account to be shown
  3. This textbox is designated to display the name of a pending user.
  4. This textbox is designated to display the course and year of a pending user.
  5. This textbox is designated to display the student number of a pending user.
  6. This button approves the selected pending user and gives it access.
  7. This button declines the selected pending user and denies it access.
  8. This button redirects the admin user back to the main menu.

Teacher Panel

This panel is exclusive to teachers and is used to add, update or delete questions.

  1. This button redirects the teacher to the window for adding questions.
  2. This button redirects the teacher to the window for updating questions.
  3. This button redirects the teacher to the window for deleting questions.
  4. This button redirects the teacher back to the main menu interface

Student Panel (Main Game)

This is the main game interface for world exploration, quests, and lesson delivery.

  1. The red bar is the player's health bar while the blue indicates the user's mana.
  2. This yellow bar indicates the player's progress in the current map.
  3. This overlay instructs users to press Tab to access quests.
  4. This overlay instructs users to press M to access the map.
  5. This overlay instructs users to press Esc to access lessons.
  6. This overlay shows the current equipped weapon of the player.
  7. This overlay shows the current magic the player has in slot.

Quest Log

The quest log is where you can review the active quests you need to complete within the game.

  1. This is where you discover the primary storyline quest in the game.
  2. This is where you locate the side quests in the game.
  3. This is where you access detailed descriptions of each quest.

Lessons

Completing the quest allows you to acquire the lesson.

  1. This is where you can view lesson topics for review.

Lesson Log

The Lesson Log is where you can access and review all the lessons you have acquired.

  1. Here you can choose the specific lesson you wish to access.
  2. You can choose which topics within the lesson you would like to explore.

Review Question Interface

To give the student user a chance to review the terms learned in one lesson.

  1. This line of text shows the current question.
  2. These numbered texts show the choices for the current question. The player can press 1, 2, 3, or 4 to select an answer.

Assessment Interface (Boss Battle)

This happens when lessons included in one grading period has been learned. This is used to assess the student's learning.

  1. This bar shows Tom's health. It decreases when the player's answer is wrong.
  2. This bar shows boss's health. It decreases when the player's answer is right.
  3. This line of text shows the current question.
  4. These numbered texts show the choices for the current question.
